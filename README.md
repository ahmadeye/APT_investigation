# Cybersecurity Incident Response & Digital Forensics Project

## Overview

This project demonstrates a complete cybersecurity incident response and digital forensics workflow based on a ransomware attack scenario. It covers attack analysis, threat mapping, forensic evidence collection, detection engineering, threat hunting, Indicators of Compromise (IoCs), and incident response following industry best practices.

The project is based on frameworks such as:

- Cyber Kill Chain
- MITRE ATT&CK Framework
- RFC 3227 (Order of Volatility)
- NIST SP 800-61 Rev. 2 (Computer Security Incident Handling Guide)

---

## Project Contents

### Part 1 & 2 – Attack Analysis
- Cyber Kill Chain mapping
- MITRE ATT&CK TTP Mapping
- Technical justification for each attack stage

### Part 3 – Digital Forensics
- Order of Volatility (OOV)
- Forensic acquisition strategy
- RAM acquisition strategy
- Disk imaging process
- Evidence integrity using MD5/SHA-256 hashes

### Part 4 – Detection Engineering
- Five Sigma detection rules
- One YARA rule
- Detection logic and explanations

### Part 5 – Threat Hunting
- Splunk SPL queries
- Microsoft Sentinel (KQL) queries
- Detection of:
  - Privilege Escalation
  - Lateral Movement
  - Data Exfiltration

### Part 6 – Indicators of Compromise (IoCs)
- Host-based IoCs
- Network-based IoCs
- Detection methodologies

### Part 7 – Incident Response
Following the NIST SP 800-61 framework:
- Containment
- Eradication
- Recovery
- Enterprise Hardening
